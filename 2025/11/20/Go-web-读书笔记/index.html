Go web 读书笔记

发表于2025-11-20|更新于2025-11-20|读书笔记
|浏览量:

总内容

本书很多都是一些老掉牙的东西，net/http包的基本web应用，cookie的设置，TLS证书和配置，HTTPS，怎么说呢，虽然很底层，但是感觉挺好用的，起码学到了很多东西，深度的刨析goweb的底层原理和就是没给我们看底层代码，但其实该写的都写了还有就是测试，挺有意思的，go模板可能就是最有价值的内容

第一章:

给我们普及了HTTP报文和url的概念，起码基础打牢了，并且叫我们写出了最简单的go web 程序

第二章：

给我们手搓了一个基于原生net/http包的goweb程序，并且给我们粗略了讲了一下后面要学一些什么东西，cookie，模板。并且补足了我对底层使用net/http包的不熟悉

第三章:

给我们介绍了一下怎么创建cookie，和TLS证书，科普了一下https和http的区别，并且告知cookie的神奇用法–短暂闪回

第四章：

介绍基础http包的内容，首先讲述了一下http.writerrespond(好像打错字了),这个东西负责把东西写入响应体里面，接下来就是响应体（忘记了怎么写了），反正本质上是一个结构体，有head和body，可以用指针调用，本质上和http的内容差不多，底层很好理解，因为看了这个，我才更好的理解了如何向ai网站构建响应体发送请求又收到响应

第五章：

本书知识精华：GO语言模板，本质上就是把html文件放到服务端渲染传递给前端，有四大行为，条件，循环，行动，模板，条件就是if else判断，循环就是range渲染，行动就是with把以后的某一个数据全改了，（相当于变量修改），模板就是用来定义模板，html模板也可以包含模板layout，但是这个好像被gin的template取代了，gin.HTML(“文件名”，/*template)更好用，感觉都在学老掉牙的东西，不过理解底层可以帮助我们学框架更爽，看代码看懂嘻嘻，还有就是模板的优点，可以防止xss攻击

第六章：

我靠学了个啥？我忘了，好像是http的基本语法吧，处理器和处理器函数的区别，SeverHTTP模式，多路复用器的妙用，还有就是就是处理器套处理器减少代码的耦合度，共用处理器数据，现在被gin中间件取代里

第七章:

通道的小妙用，给初学者讲解了一下并发编程，很基础，聊胜于无，不过后面给我们演示了怎么编写一个马赛克的网站，但是我看不懂就没看了555.

第八章：

测试：三个模块：第一个：基本用法，test包，还有bench测速，我靠其他忘的差不多了，书白看了555，第二个，第三个就是现代的测试包，也忘了，55书白看了

第九章：

打包，docker，hero，微软的一个东西，我只看了docker，首先docker -t build 构建镜像文件，然后本地就是 docker run 推送doceker push
